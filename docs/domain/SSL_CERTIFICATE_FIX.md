# SSL Certificate Fix for resources.sageoutdooradvisory.com

## üö® Root Cause Identified

**Error:** `NET::ERR_CERT_COMMON_NAME_INVALID`  
**Symptom:** Chrome shows "Your connection is not private" warning  
**Certificate Shows:** `*.bluehost.com` (wildcard certificate for Bluehost)  
**Domain Being Accessed:** `resources.sageoutdooradvisory.com`

### The Problem

The SSL certificate error is happening because **your DNS is pointing to Bluehost instead of Vercel**. 

When you access `resources.sageoutdooradvisory.com`, the domain resolves to Bluehost's servers, which have a certificate for `*.bluehost.com`. This certificate doesn't match your domain, causing the error.

### The Solution (Keep Bluehost Nameservers)

**‚úÖ You can keep Bluehost nameservers!** Just add a CNAME record in Bluehost pointing to Vercel:

1. Get CNAME target from Vercel (Project Settings ‚Üí Domains)
2. In Bluehost ‚Üí Domains ‚Üí `sageoutdooradvisory.com` ‚Üí DNS tab
3. Add CNAME record: `resources` ‚Üí `[Vercel's CNAME target]`
4. Wait 15-30 minutes for DNS propagation
5. SSL certificate will auto-generate on Vercel

**No need to change nameservers!** See detailed steps below.

## Solution: Configure Domain in Vercel

This is a **Vercel infrastructure issue**, not a code issue. The domain needs to be properly added and verified in Vercel.

1. **Log into Vercel Dashboard**

2. **Go to Project Settings ‚Üí Domains**
   - Click on your project
   - Go to **Settings** tab
   - Click **Domains** in the left sidebar

3. **Add the Domain**
   - Click **Add Domain** button
   - Enter: `resources.sageoutdooradvisory.com`
   - Click **Add**

4. **Follow DNS Instructions**

### Step 2: Configure DNS Records in Bluehost (Keep Bluehost Nameservers)

**‚úÖ You can keep Bluehost nameservers!** You just need to add a CNAME record in Bluehost pointing to Vercel.

**‚ö†Ô∏è DO NOT Enable SSL in Bluehost:**
- You don't need SSL certificates in Bluehost for this subdomain
- Since the CNAME points to Vercel, Vercel handles all SSL automatically
- Bluehost SSL settings are for websites hosted ON Bluehost, not for domains pointing elsewhere
- Enabling SSL in Bluehost could cause conflicts or confusion

#### Get Vercel's CNAME Target First

1. **Go to Vercel ‚Üí Project Settings ‚Üí Domains**
2. **Click on `resources.sageoutdooradvisory.com`**
3. **Copy the CNAME target value** (e.g., `cname.vercel-dns.com` or your project's Vercel URL)
4. **Keep this value handy** - you'll need it in Bluehost

#### Add CNAME Record in Bluehost

1. **Log into Bluehost**
   - Go to [bluehost.com](https://bluehost.com) and sign in
   - Navigate to your account dashboard

2. **Go to Domain Management**
   - Click **"Domains"** in the left sidebar
   - Find and click on **`sageoutdooradvisory.com`**
   - Click on the **"DNS"** tab

3. **Add CNAME Record**
   - Scroll to the **"Manage Advanced DNS Records"** section
   - Click **"+ ADD RECORD"** button
   - Fill in the form:
     - **Type:** Select `CNAME` from dropdown
     - **Host Record:** Enter `resources` (just the subdomain name, not the full domain)
     - **Point To:** Paste the Vercel CNAME target you copied earlier
       - Example: `cname.vercel-dns.com` or `[your-project].vercel.app`
     - **TTL:** Leave as default (usually 14400 or 3600)
   - Click **"Save"** or **"Add Record"**

4. **Remove Conflicting Records (If Any)**
   - If there's an existing **A record** for `resources`, delete it
   - If there's an existing **CNAME record** for `resources` pointing to Bluehost, delete it
   - CNAME records take precedence, so only one should exist

5. **Verify the Record**
   - You should see a new CNAME record in the DNS table:
     - Type: `CNAME`
     - Host Record: `resources`
     - Point To: `[your-vercel-target]`
   - The record should be active

#### Important Notes for Bluehost

- **Keep Bluehost Nameservers:** You don't need to change nameservers - keep them as Bluehost's
- **DNS Propagation:** Changes can take 15-30 minutes (up to 24-48 hours globally)
- **TTL:** Lower TTL values (like 3600) make changes propagate faster
- **Wait Time:** Bluehost may show a note that changes take 24-48 hours, but usually it's faster

#### Verify DNS is Working

1. **Check DNS Propagation**
   - Use [whatsmydns.net](https://www.whatsmydns.net) or [dnschecker.org](https://dnschecker.org/)
   - Enter: `resources.sageoutdooradvisory.com`
   - Should resolve to Vercel's IP addresses (not Bluehost's)

2. **Command Line Check**
   - Run: `nslookup resources.sageoutdooradvisory.com`
   - Should show Vercel's servers, not Bluehost

3. **Wait for Vercel to Detect**
   - Once DNS propagates, Vercel will detect the change
   - SSL certificate will auto-generate (takes 5-60 minutes)

## Frequently Asked Questions

### Q: Do I need to enable SSL certificate in Bluehost for the subdomain?

**A: NO!** You don't need SSL in Bluehost at all.

**Explanation:**
- Your subdomain (`resources.sageoutdooradvisory.com`) uses a CNAME record pointing to Vercel
- This means Vercel hosts your website, not Bluehost
- Vercel automatically generates and manages SSL certificates for all domains added to your project
- Bluehost is only managing DNS records (the CNAME), not hosting your site
- SSL certificates in Bluehost are only for websites actually hosted ON Bluehost's servers

**What happens:**
1. User visits `https://resources.sageoutdooradvisory.com`
2. DNS (managed by Bluehost) resolves to Vercel's servers
3. Vercel serves your website with Vercel's SSL certificate
4. Bluehost is never involved in serving your website

**You only need:**
- ‚úÖ CNAME record in Bluehost (you already have this)
- ‚úÖ Domain added in Vercel (you already have this)
- ‚úÖ Wait for Vercel to generate SSL (automatic, 5-60 minutes)

### Q: Why is Bluehost's SSL certificate showing (`*.bluehost.com`)?

**A:** This happens when:
- DNS is resolving to Bluehost's servers instead of Vercel's
- There's a conflicting A record pointing to Bluehost
- DNS hasn't fully propagated yet
- The CNAME record isn't working correctly

**Solution:** Check for conflicting A records and verify DNS propagation.

### Q: Can I use Bluehost's SSL certificate instead?

**A:** No, and you don't need to. Vercel automatically provides free SSL certificates via Let's Encrypt. These are:
- Free
- Automatically renewed
- Managed by Vercel
- More reliable than managing certificates manually

### Q: What if I see SSL options in Bluehost?

**A:** Ignore them. Those SSL options are for websites hosted on Bluehost's servers. Since your subdomain points to Vercel, those settings don't apply to your subdomain.
